## 2.1 — Переменные и типы данных

### Концепция

В Java переменные строго типизированы: тип объявляется явно и не меняется.
Примитивные типы хранят значение напрямую, ссылочные — адрес объекта в куче (heap).

### Примитивные типы в коде

| Тип | Размер | Диапазон | Где используется в игре |
|-----|--------|----------|------------------------|
| `byte` | 8 бит | −128..127 | `Game.java:247` — побитовые флаги статусов |
| `int` | 32 бита | ≈ ±2 млрд | `Game.java:175` — `MAX_ENEMIES`, здоровье, атака, защита |
| `long` | 64 бита | ≈ ±9×10¹⁸ | `Game.java:160` — `serialVersionUID`, счёт игры |
| `double` | 64 бита | IEEE 754 | `GameCharacter.java:363` — множитель атаки |
| `boolean` | 1 бит | true/false | `Game.java:1617` — флаг получения урона, результат боя |
| `char` | 16 бит | Unicode | `Game.java:2731` — ранг (S, A, B, C, D) |

#### `Game.java`, строки 162–257 — объявление полей

```java
// int — примитивный целочисленный тип (32 бита, от -2^31 до 2^31-1).

// String — ССЫЛОЧНЫЙ тип (объект). Неизменяемый (immutable).
// Каждое изменение строки создаёт НОВЫЙ объект.

// byte — 8 бит, достаточно для 8 булевых флагов в одной переменной.
private byte statusFlags = 0;

// boolean — примитивный тип: только true или false.

```

#### `Game.java`, строка 2731 — переменная типа `char`

```java

    rank = 'S';
```

### Ссылочные типы


## 2.3 — Арифметические операторы

### Концепция

Java поддерживает стандартные арифметические операторы. Важно помнить об
**целочисленном делении** (int / int = int, дробная часть отбрасывается) и
**приведении типов** при смешивании int и double.

### Операторы в коде

| Оператор | Смысл | Файл : строка |
|----------|-------|---------------|
| `+` | Сложение, конкатенация строк | `Game.java:534`, `GameCharacter.java:618` |
| `-` | Вычитание | `GameCharacter.java:420` — `damage - defense` |
| `*` | Умножение | `GameCharacter.java:363` — `attack * multiplier` |
| `/` | Деление (целое для int!) | `Game.java:2715` — `(double) enemiesDefeated / MAX_ENEMIES` |
| `%` | Остаток от деления | В коде игры не используется как арифметический оператор (только в строках формата `printf`). Классический пример: `n % 2 == 0` — проверка чётности. |
| `++` | Инкремент (+1) | `Game.java:1034` — `enemiesDefeated++` |
| `+=` | Сложение с присваиванием | `GameCharacter.java:469` — `experience += exp` |
| `*=` | Умножение с присваиванием | `Game.java:2750` — `bonus *= hero.getLevel()` |

#### `Game.java`, строки 2715–2717 — приведение типов при делении

```java
// ЦЕЛОЧИСЛЕННОЕ ДЕЛЕНИЕ: 3 / 5 = 0 (дробная часть отброшена — это баг!)
// Чтобы получить 0.6, нужно привести один операнд к double:
double winRate = (double) enemiesDefeated / MAX_ENEMIES * 100;

// (int) winRate — сужающее приведение double → int.
// Дробная часть ОТБРАСЫВАЕТСЯ: 60.7 → 60 (НЕ округляется!).
int winRateInt = (int) winRate;

// (long) totalDamageDealt — расширяющее приведение int → long.
// Нужно, чтобы int * int не переполнился (max int ≈ 2 млрд).
long totalScore = (long) totalDamageDealt * enemiesDefeated;
```

#### `GameCharacter.java`, строка 420 — `Math.max()`

```java
// Math.max(a, b) — стандартная библиотечная функция, возвращает большее из двух чисел.
```

## 2.4 — Поразрядные операторы

### Концепция

Побитовые (поразрядные) операторы работают с каждым битом числа отдельно.
В игре используются для компактного хранения статусных эффектов (яд, оглушение, щит, ярость)
в одном поле типа `byte` — это 8 независимых флагов в одной переменной.

### Побитовые операторы в коде

| Оператор | Действие | Файл : строка |
|----------|----------|---------------|
| `<<` | Сдвиг влево (умножение на 2ⁿ) | `Game.java:255–257` — создание масок |
| `&` | Побитовое И (проверка флага) | `Game.java:1800` — `(statusFlags & STUNNED) == 0` |
| `\|` | Побитовое ИЛИ (установка флага) | `Game.java:2269` — `statusFlags \|= STUNNED` |
| `~` | Побитовое НЕ (инверсия для снятия флага) | `Game.java:1804` — `~STUNNED` |
| `&=` | И с присваиванием (снятие флага) | `Game.java:1804` — `statusFlags &= ~STUNNED` |
| `\|=` | ИЛИ с присваиванием (установка флага) | `Game.java:2269` — `statusFlags \|= STUNNED`, `Game.java:2473` — `\|= SHIELDED`, `Game.java:2584` — `\|= POISONED` |
| `^=` | XOR с присваиванием (переключение флага) | `Game.java:2509` — `statusFlags ^= ENRAGED` (toggle: вкл/выкл) |

#### `Game.java`, строки 231–257 — создание масок и работа с флагами

```java
// byte — 8 бит = 8 независимых флагов в одной переменной.
private byte statusFlags = 0;  // 0 = 0b00000000 — все флаги выключены

// Создание масок через сдвиг влево (<<):
// 1       = 0b00000001  — бит 0 (яд)
// 1 << 1  = 0b00000010  — бит 1 (оглушение)
// 1 << 2  = 0b00000100  — бит 2 (щит)
// 1 << 3  = 0b00001000  — бит 3 (ярость)
private static final byte POISONED = 1;
private static final byte STUNNED  = 1 << 1;
private static final byte SHIELDED = 1 << 2;
private static final byte ENRAGED  = 1 << 3;
```

#### `Game.java`, строки 1800–1804 — чтение и снятие флага

```java
// Проверка флага: побитовое И с маской.
// Если бит STUNNED установлен — результат != 0 (оглушён, пропускает ход).
// Если бит не установлен — результат == 0 (ходит нормально).
if ((statusFlags & STUNNED) == 0) {
    enemyTurn(enemy);
} else {
    System.out.println(enemy.getName() + " оглушён и пропускает ход!");
    // Снятие флага: &= ~STUNNED
    // ~STUNNED инвертирует все биты маски: 0b00000010 → 0b11111101
    // & оставляет все биты кроме бита 1 без изменений.
    statusFlags &= ~STUNNED;
}
```

#### `Game.java`, строки 2469–2516 — `|=` (установка флага) и `^=` (переключение флага)

```java
// |= (ИЛИ с присваиванием) — устанавливает бит в 1.
// statusFlags |= SHIELDED  →  устанавливает бит 2 (щит).
// Если бит уже был 1 — ничего не изменится (идемпотентная операция).
// Другие биты НЕ затронуты — ключевое свойство побитового OR.
statusFlags |= SHIELDED;   // Game.java:2473

// ^= (XOR с присваиванием) — ПЕРЕКЛЮЧАЕТ бит (toggle):
//   Если ENRAGED был 0 → станет 1 (ярость ВКЛЮЧАЕТСЯ).
//   Если ENRAGED был 1 → станет 0 (ярость ВЫКЛЮЧАЕТСЯ).
// Удобно для флагов-переключателей: одна операция заменяет if-else.
statusFlags ^= ENRAGED;    // Game.java:2509
```

Аналогично `|=` используется в `Game.java:2269` (`|= STUNNED`) и `Game.java:2584` (`|= POISONED`).

#### `Game.java`, строка 1839–1856 — флаг POISONED в конце раунда

```java
// Проверяем бит POISONED: (statusFlags & POISONED) != 0
if ((statusFlags & POISONED) != 0 && hero.isAlive()) {
    hero.takeDamage(3);
    // Math.random() — случайное double из [0.0, 1.0). Вероятность < 0.4 = 40%.
    if (Math.random() < 0.4) {
        statusFlags &= ~POISONED;  // Снимаем яд тем же способом
    }
}
```

#### `Game.java`, строки 709–716 — `switch`-выражение (Java 14+)

```java
// switch-ВЫРАЖЕНИЕ — новый синтаксис (с Java 14, стандарт с Java 16).
// Отличия от старого switch-оператора:
//   1. Возвращает значение — можно присвоить переменной.
//   2. -> вместо : — НЕТ «проваливания» (fall-through). break не нужен.
//   3. Компилятор требует обработать ВСЕ варианты.
hero = switch (choice) {
    case 1 -> new Warrior(name);
    case 2 -> new Mage(name);
    case 3 -> new Archer(name);
    // default — обязателен для int: компилятор не знает все возможные значения.
    // Для sealed-иерархий или enum с полным покрытием — не нужен.
    default -> new Warrior(name);
};
```

#### `Game.java`, строка 1996 — switch-выражение возвращает строку

```java
// Ещё один пример switch-выражения: результат присваивается String.
// Каждая ветка -> возвращает строку напрямую (без return, без break).
String actionName = switch (action) {
    case 1 -> "Обычная атака";
```

```java
switch (action) {
```

## 2.6 — Циклы

### Концепция

В Java четыре вида цикла. Выбор определяется задачей:
- `for` — когда число итераций известно заранее

- `do-while` — когда тело нужно выполнить хотя бы раз
- `for-each` — когда перебираем коллекцию или массив без доступа к индексу

| `break` | `Game.java:1097` | Досрочный выход из цикла |

#### `Game.java`, строки 686–693 — `do-while`

```java
// do-while — выполняет тело ХОТЯ БЫ ОДИН РАЗ, затем проверяет условие.
// Идеален для ввода: нужно спросить хотя бы раз, потом валидировать.
// Отличие от while: while проверяет ДО тела (может не выполниться ни разу).

do {
    System.out.println("Выберите класс: 1. Воин  2. Маг  3. Лучник");
```

```java
private int readInt() {

        return Integer.parseInt(scanner.nextLine().trim());
```

```java
        return -1;
    } finally {

        // Типично используется для освобождения ресурсов.
    }
}
```

## Частые вопросы (FAQ)

**Q: Чем `String` отличается от `char`?**
`char` — один символ (`'A'`). `String` — строка, объект, содержит ноль и более символов (`"Артур"`).

**Q: Почему `3 / 5 == 0`, а не `0.6`?**
Оба операнда — `int`, значит результат тоже `int` (целочисленное деление). Дробная часть отбрасывается. Чтобы получить `0.6`, нужно привести один операнд: `(double)3 / 5`.

**Q: В чём разница между `while` и `do-while`?**
`while` проверяет условие ДО выполнения тела — может не выполниться ни разу. `do-while` выполняет тело ХОТЯ БЫ РАЗ, потом проверяет условие. Используй `do-while` для ввода с валидацией.

**Q: Что такое `++` и чем `i++` отличается от `++i`?**
Оба увеличивают переменную на 1. Разница проявляется при использовании в выражении: `i++` возвращает старое значение, `++i` — новое. В цикле `for` разницы нет.

**Q: Когда использовать `for-each`, а когда `for` со счётчиком?**
`for-each` — когда нужны только значения элементов, без индекса. `for` со счётчиком — когда нужен индекс для доступа к соседним элементам или для нескольких коллекций параллельно.

**Q: Что будет, если выйти за границу массива (`array[array.length]`)?**
JVM бросит `ArrayIndexOutOfBoundsException`. Максимальный валидный индекс — `array.length - 1`.
